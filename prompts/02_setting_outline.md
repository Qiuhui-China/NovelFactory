## 番茄短故事结构规律（从真实爆款总结）

1. **开头即正文**：没有"简介"，第一句话就是故事，就是钩子。
2. **数字分段**：用"1." "2." "3."分场景，不用章节标题。每段长度不固定，按场景切分。
3. **一句一段是常态**：60%的段落只有一句话。对话单独成段。
4. **对话占60-70%**：大量对话连续出现，不写"他说""她道"。用动作穿插区分说话人。
5. **内心独白=脑内弹幕**：极短、情绪裸露、标点暴力。"呵。""急！慌！""蠢。且毫无敬畏。"
6. **打脸靠反应出爽感**：不长篇描写打脸过程，靠对手/围观群众的反应让读者爽。
7. **结尾收在轻盈的对话上**：不要宏大收尾，一句俏皮话或温暖对话就结束。

## 关键规则

1. **主角必须是女性（大女主）**。故事以女主第一人称视角展开，核心爽感来自女主的逆袭/反击/打脸，绝对不能以男性角色作为主角。角色表中第一个必须是女主（role="女主"），不得出现 role="主角"。
2. **有名有姓的核心角色不超过4个**。但可以有不限数量的"龙套"——没有名字的功能性角色，如管家、路人、同事、围观群众等。龙套作用：点缀场景真实感、推动剧情（传话、通风报信）、制造群体反应（窃窃私语、鼓掌、起哄）。龙套不需写进角色表，但 events 里可以写"围观同事议论纷纷"或"管家来报说客人到了"。
3. **speech_style 要具体到能直接模仿**："喜欢用'呵'开头冷嘲，生气时说话变短，心软时用省略号"
4. **每个角色必须有 secret**——即使是配角。秘密是制造反转的弹药。
5. **outline 的 events 必须写具体事件**，不能写"继续发展"。
6. **打脸至少3次**，烈度递增，分布在全文不同位置。
7. **"您"只在真正表达尊敬时使用**（晚辈→长辈、下属→上级），禁止用"您"做嘲讽/内心旁白/挑衅语气。对抗场景中统一用"你"，不论对方身份。hook_ending 和 events 中如有对话，须遵守此规则。
8. **每个场景段必须有隐含的时间、地点、在场人物**。不需要写"时间：XX 地点：XX"，但读者必须能从对话内容、动作描写或环境细节中自然推断出场景。大纲的 events 描述中，每个场景段标注一下主要地点（已在 outline 的 location 字段体现）。
9. **场景切换不能跳跃太大**。如果上一段在家里，下一段突然在公司，中间至少需要一句过渡（哪怕是"第二天一早"这样的时间词）。大纲里标注场景切换点。
